Lignes d'une classe : toutes les lignes à l'intérieur de {}, même si juste }
Lignes d'une méthode : lignes à l'intérieur de la méthode (n'inclut pas le nom de la méthode et })
Un commentaire qui est situé avant une méthode doit être collé à la méthode sans espace vide entre pour être considéré comme un commentaire de la méthode. Les commentaires d'une méthode sont situés directement avant la méthode, ou à l'intérieur des {}.
@override : commentaire
import et package : pas une ligne de code
On peut avoir un commentaire d'une seule ligne après le } de fin de classe et de ifn de méthode

On suppose que le programme java suit les conventions. Par exemple, public void test(int a) {for(i=1; i <100; i++) a++;} serait écrit comme :
public void test(int a) {
	for(i=1; i <100; i++) a++;
}

Enum considéré comme une classe s'il est à l'extérieur d'une classe. S'il est dans une classe, il sera considéré comme une méthode.
Si le enum est sur une seule ligne, la ligne sera compté dans le nombre de ligne de la classe (ou de la méthode) (=1)

* Complexité cyclomatique de McCab :
V(F) = 1 + d où d est le nombre de noeuds prédicats
Noeuds prédicats = noeud avec au moins 2 sorties
** if a then A (conditional structure) + 1
** if a then A else B (conditional structure) +1
** if a then A else if B then B else C +2
** switch case (+ 1 per case or default block) +1 par case
** while a do A (iterative structure) +1
** repeat a until A (iterative structure) +1
(boolean condition) (throw, try, catch) ignoré
